## 아이디어 배경

이직을 준비하면서 느꼈다. "내가 가진 스킬"과 "목표 직무가 요구하는 스킬" 사이의 갭을 정확히 아는 사람이 없다. 채용공고를 하나하나 읽으며 비교하는 건 비효율적이고, 자기 객관화는 더 어렵다. AI가 이걸 해주면 어떨까?

## 기획 과정

### 수익 모델
- 기본 진단: 무료 (1회)
- 인프런 어필리에이트: 스킬갭에 맞는 강의 추천 시 커미션
- 상세 로드맵: 유료 (미구현)

### 핵심 플로우
1. 이력서 텍스트 붙여넣기
2. 목표 직무 선택 (PM, 마케터, 개발자 등)
3. AI가 스킬갭 분석 + 학습 로드맵 생성
4. 추천 강의 링크 (인프런 어필리에이트)

### 의사결정
처음에는 이력서 파일 업로드(PDF)를 지원하려 했는데, 파싱 정확도 문제로 텍스트 붙여넣기로 단순화했다. MVP에서는 "되는 것"이 중요하니까.

## 커밋 로그

### 2026-05-01: 아이디어 구체화
이직 준비하면서 채용공고 10개를 비교 분석했는데, 같은 "PM" 직무도 회사마다 요구 스킬이 다르다는 걸 체감. 이걸 자동화하고 싶었다.

### 2026-05-07: 프롬프트 설계
Claude API로 이력서 분석 프롬프트를 설계. 핵심은 "보유 스킬 추출 → 목표 직무 요구 스킬 매칭 → 갭 분석 → 우선순위 정렬". 프롬프트가 너무 길어져서 단계별로 분리했다.

### 2026-05-12: UI 구현
Next.js + Tailwind로 3단계 위저드 UI. 결과 페이지에서 스킬을 "보유/부족/강화필요" 3단계로 시각화.

### 2026-05-18: 인프런 어필리에이트 연동
스킬갭 결과에서 "부족" 항목마다 인프런 강의 검색 링크를 자동 생성. 어필리에이트 코드 포함.

### 2026-05-22: MVP 완성
배포 직전까지 완성. 테스트로 내 이력서를 넣어봤는데 생각보다 정확해서 놀랐다.

## 회고

프롬프트 엔지니어링이 80%였다. "스킬 추출"과 "갭 분석"을 한 번에 하면 품질이 떨어져서, 2단계로 나눠야 했다. 수익화는 어필리에이트가 가장 자연스러운 구조인데, 실제로 전환이 일어날지는 운영해봐야 안다.
